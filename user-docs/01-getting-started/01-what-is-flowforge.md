# What is FlowForge?

FlowForge is an AI-assisted API testing platform that helps QA teams import API specifications, generate test scenarios, execute tests against live endpoints, and track results — all from a single web application.

## Who is FlowForge for?

FlowForge is built for QA teams that need to test REST APIs. Whether you're a QA engineer writing daily tests or a QA manager overseeing test coverage across multiple projects, FlowForge streamlines the entire workflow from API spec to verified test results.

## What can you do with FlowForge?

### Import and manage API specifications

Upload OpenAPI/Swagger spec files or import them directly from a URL. FlowForge splits large specs into per-endpoint reference files, organizes them into version folders, and keeps them in sync when the API evolves.

### Generate test ideas with AI

Select a group of API endpoints and let AI analyze the specs to suggest meaningful test scenarios — happy paths, error cases, edge cases, and multi-step workflows that test entity dependencies.

### Author flow definitions

Each test scenario is defined as a **flow** — an XML document that describes the sequence of API calls, assertions, and data captures. Flows can be generated by AI from ideas, written manually, or refined through an interactive chat with the AI assistant.

### Run tests against live endpoints

Connect FlowForge to your API endpoint (supports OAuth 2.0, bearer tokens, API keys, basic auth, and cookie-based auth). Run individual scenarios or entire batches, with real-time step-by-step results showing pass/fail status, response data, and assertion outcomes.

### Debug and diagnose failures

When a test step fails, FlowForge's AI diagnosis analyzes the failure against the API spec, identifies the root cause, and can automatically fix common issues like incorrect field names or missing required parameters.

### Track results over time

Every test run is persisted with full step-level detail. Compare runs side by side, spot regressions, and build confidence in your API's reliability.

## How FlowForge works — the big picture

The typical workflow follows these stages:

1. **Import specs** — Bring your OpenAPI/Swagger specification into FlowForge via the Spec Manager
2. **Generate ideas** — AI analyzes your endpoints and suggests test scenarios
3. **Generate flows** — AI creates detailed flow XML definitions from selected ideas
4. **Create scenarios** — Register flows as runnable test scenarios in the Scenario Manager
5. **Connect & run** — Set up an API connection and execute your scenarios against a live endpoint
6. **Review results** — Inspect step-by-step outcomes, diagnose failures, iterate

## Key features at a glance

| Feature | Description |
|---|---|
| **OpenAPI import** | Import from URL or upload files; auto-splits into per-endpoint specs |
| **AI idea generation** | Context-aware test scenario suggestions from your API specs |
| **Flow XML authoring** | AI-generated or hand-crafted test definitions with full XML schema |
| **Interactive flow chat** | Refine flows through conversation with the AI assistant |
| **Multi-auth support** | OAuth 2.0, bearer token, API key, basic auth, cookie auth |
| **Step-level results** | Pass/fail per step with response body, headers, and assertion detail |
| **AI diagnosis** | Automated root-cause analysis and fix suggestions for failures |
| **Run history** | Persistent test run records with comparison capability |
| **Role-based access** | 5-tier permission hierarchy from Member to Super Owner |
| **Project variables** | Shared key-value pairs for dynamic test data across flows |
| **Public API** | Run scenarios programmatically and integrate with CI/CD pipelines |
